Linux commands

1. Opening a terminal

1.1. A sense of location

1.1.1. Comanda pwd ne spune exact care este directorul curent, (în ce fișier ne aflăm).
- începând cu rădăcina; rădăcina este baza principală a unui sistem Linux. Rădăcina este indicată printr-o linie oblică (/). 
                        pwd

pwd = print working directory


1.1.2. Comanda cd este folosită pentru a merge într-un director.
                        cd /downloads
Comanda cd schimba directorul de lucru

cd = change directory

Din directorul rădăcină, următoarea comandă vă va muta în directorul „acasă"
                        cd home
                        pwp
O scurtătură rapidă pentru a ajunge la directorul "acasă" este tastând doar cd 
                        cd
                        pwp
Pentru a merge la directorul părinte, în acest caz înapoi la „/”, folosiți sintaxa specială a două puncte (..)
                        cd ..
                        pwp
Pentru a vă deplasa prin mai multe niveluri de directoare părinți, puteți utiliza (..) de mai multe ori
                        cd ../..
                        pwp
Dacă dorim să mergem direct din directorul nostru principal în directorul „etc” (care se află direct în rădăcina sistemului de fișiere), putem folosi:
                        cd
                        pwp
                        cd ../..etc
                        pwp

1.2. Relative and absolute paths

Orice cale care începe / înainte este o cale absolută.

Comanda ls ne afișează conținutul directorului curent. Putem vedea toate fișierele ascunse.
                    ls

ls = list their contents


2. Creating folders and files

2.1. Comanda "mkdir" este utilizată atunci când trebuie să creați un folder sau un director.

                        mkidr /tmp/tutorial

                        cd /tmp/tutorial

mkidr = make directory

Observați utilizarea unei căi absolute, pentru a asigura crearea directorului tutorial în interiorul lui / tmp. Fără / la început, comanda mkdir ar încerca să găsească un director tmp în directorul de lucru curent, apoi să încerce să creeze un director tutorial în interiorul acestuia. Dacă nu s-a găsit un director tmp, comanda nu ar reuși.
2.2. Creează câteva subdirectoare:
                        mkdir dir1 dir2 dir3
Până acum, am văzut doar comenzi care funcționează pe cont propriu (cd, pwd) sau care au un singur articol ulterior (cd /, cd ~ / desktop). Dar de această dată am adăugat trei lucruri după comanda mkdir. Aceste lucruri sunt denumite parametri sau argumente.Comanda mkdir se așteaptă la cel puțin un argument, în timp ce comanda cd poate funcționa cu zero sau unul, dar nu mai mult.

Observați că mkdir a creat toate folderele dintr-un singur director. Nu a creat dir3 în interiorul dir2 în interiorul dir1 sau în orice altă structură. Dar, uneori, este util să poți face exact asta, iar mkdir are o cale:

2.3. Creează mai multe directoare doar trecându-le ca argumente de separare la comanda mkdir.
                        mkdir -p dir4/dir5/dir6

De această dată veți vedea că doar dir4 a fost adăugat la listă, deoarece dir5 este în interiorul ei, iar dir6 este în interiorul acesteia.

„-p” pe care l-am folosit se numește option sau switch (în acest caz înseamnă „creați și directoarele părinte”). Opțiunile sunt utilizate pentru a modifica modul în care operează o comandă, permițând unei singure comenzi să se comporte într-o varietate de moduri diferite.
2.4. Creează un director cu un spațiu în nume folosind "", '' sau \
                        mkdir "folder 1"
                        mkdir 'folder 2'
                        mkdir folder\ 3
                        "folder 4" "folder 5"
                        -p "folder 6"/"folder 7"
                        ls
3. Moving and manipulating files

3.1. Comanda mv redenumesțe sau mută fișierele

                            mv
                        
                            mv copytest.txt newcopytest.txt

mv = rename or move files

3.2. Deleting files and folders

Comanda rm este folosit pentru ștergerea unui fișier complet

Comanda rmdir este folosită pentru ștergerea unui director gol.

                            rm -r folder_6

GitHub commands

1. Check the status of the repository / git status

Utilizați comanda status pentru a verifica starea curentă a depozitului.
Folosesc comanda git status pentru a continua monitoriza stările atât a directorului de lucru, cât și a depozitului.
                                        git status
Comanda verifică starea și raportează că nu există nimic de commit (nu exist[ modificări de înregistrat).

2. Add files / git add

Comanda add se folosește pentru a adăuga un fișier individual sau grupuri de fișiere.
Pentru a adăuga fișierul index.html pe care l-am modificat folosesc:
                    git add index.html
Pentru a adăuga toate fișierele pe care le-am editat, în același timp, pot utiliza:
                    git add .

3. Commit files / git commit -m

Comanda commit necesită un mesaj de validare care descrie modificările pe care le-am făcut în acel fișier.
Pot crea un mesaj scurt folosind indicatorul -m. Mesajul ar trebui să exprime exact ce s-a schimbat și de ce.
                    git commit -m "Editing the index to add the CV in Romanian language"

4. Push changes to GitHub / git push

Până în prezent, am modificat doar copia noastră locală a depozitului. Pentru a adăuga modificările la fișierele mele git repo de pe computer la versiunea mea repository de pe GitHub, trebuie să folosim push.
Pot împinge modificările către GitHub cu:
                    git push
Îmi va fi solicitat apoi numele de utilizator și parola pentru GitHub.
